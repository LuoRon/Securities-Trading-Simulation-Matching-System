一、题目:证券竞价交易模拟撮合系统
二、功能需求:实现证券买卖订单的接收，加入订单队列，并按照一定的规则与未成的对手方订单进行实时撮合，将成交信息返回给对应的订单提交方，同时实时更新和揭示实时行情快照，发布逐笔成交信息。
2.1支持订单的实时撮合
    撮合系统启动时初始化证券代码，只允许对初始化范围内的证券进行交易。
    支持接收限制类订单，限制类订单是指定了标的证券，买卖方向，价格，数量等要素。
    撮合系统对订单的证券，买卖方向，价格，数量进行基本的合法性较验。
    订单成功接受后放入订单队列，撮合系统内部按“价格优先、时间优先”原则进行撮合处理，对未成交部分的订单缓存到订单队列中，与后续的新进订单撮合；撮合成交后，将订单成交信息推送给对应的订单提交方。
    支持订单的撤单处理，撤单处理后将订单从订单队列移除，不再参与后续的撮合，并反馈撤单成交响应。
　　撮合处理不考虑以下情况：
　　不考虑账户当前的实际资金，所有买入都认为资金可用足够。
　　不考虑账户当前的实际持仓，所有卖出都认为持仓可用足够。
　　（1）最高买入申报与最低卖出申报价格相同，以该价格为成交价；
　　（2）买入申报价格高于即时揭示的最低卖出申报价格时，以即时揭示的最低卖出申报价格为成交价；
　　（3）卖出申报价格低于即时揭示的最高买入申报价格时，以即时揭示的最高买入申报价格为成交价。
2.2实时行情发布
    支持以下的行情信息发布和发布特性
    a、发布实时行情快照
    b、发布实时逐笔成交明细
    c、支持行情的发布/订阅，只推送周边关注的证券代码行情

三、设计思路:将系统分为服务端和客户端，以tcp通信
　　3.1服务端:主要存储相关订单、订阅等信息，能同时跟多个客户端交互，处理多个客户的交易操作。
　　3.2客户端:每个客户端允许一个用户登录操作，类似一个app。
四、技术路线
4.1服务端
    1.利用IO复用技术epoll与线程池实现多线程的reactor模型,实现tcp通信;
    2.自定义通信协议，实现与客户端的交互;
    3.用mysql存储客户订单等信息,利用RAII机制实现了数据库连接池，减少数据库连接建立与关闭的开销。

4.2客户端
    1.以tcp与服务端交互，自定义通信协议。
    2.基于Qt5.13.2制作操作界面，实现“买入”、“卖出”、“撤销”、“订阅/发布”功能。
